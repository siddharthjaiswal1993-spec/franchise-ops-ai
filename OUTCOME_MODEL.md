# Outcome Model — AI-Native Franchise Operations GTM Strategy

---

## The Value Hypothesis

This repository documents a GTM and product strategy, not a live product. The outcome model describes the value that the described platform would create for franchise brands and multi-unit operators — and the metrics that would validate that value hypothesis.

---

## Business Metrics (For the Described Platform)

| Metric | Target | Why It Matters |
|---|---|---|
| Location health improvement | Measurable increase in average Location Health Score in 6 months | The primary product outcome |
| Risk identification lead time | 3–4 weeks earlier than manual review baseline | Earlier detection = cheaper intervention |
| Field consultant leverage | 40% more locations managed per consultant | The core MSP / franchisor efficiency metric |
| Intervention success rate | >60% of AI-recommended interventions show positive location response | Validates that the AI recommendations are actionable |

---

## Product Quality Metrics

| Metric | Target | Why It Matters |
|---|---|---|
| Location Health Score accuracy | Correlation >0.65 with 90-day location outcome | The score only matters if it predicts real outcomes |
| Recommendation acceptance rate | >55% | Field consultants must trust the system enough to act on it |
| Root cause accuracy | >70% match to field consultant assessment | Wrong root cause = wrong recommendation |
| False positive rate on risk alerts | <20% | Too many false positives and consultants stop looking |

## The Metric That Matters Most

**Field consultant recommendation acceptance rate.**

A field consultant is the most knowledgeable person in the system. If they consistently accept AI-generated next-best-action recommendations, the system has earned operational trust. If they ignore recommendations, the root cause analysis is wrong, the recommendations are misaligned with how field work actually gets done, or both.

Acceptance rate is the leading indicator. Everything else follows from it.
