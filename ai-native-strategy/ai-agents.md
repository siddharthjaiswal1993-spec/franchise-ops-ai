# AI Agents: Franchise Operational Intelligence Platform

> Documentation of 8 AI agents — purpose, triggers, actions, approval models, and success metrics.

---

## Agent Design Principles

All AI agents in the franchise operational intelligence platform operate under five non-negotiable principles:

1. **Human oversight gates.** No agent takes an action that affects a franchisee or external party without explicit human approval — or, for lower-risk automations, human override capability within a defined window.
2. **Source transparency.** Every recommendation and action generated by an agent includes the specific data that drove the decision, the confidence level, and the model version.
3. **Override capture.** When a human overrides or modifies an agent action, the reason is captured in a structured format for model improvement.
4. **Audit log completeness.** Every agent action — including proposed actions that were not approved — is logged with timestamps, rationale, and decision outcome.
5. **Graceful degradation.** If data quality is insufficient for a confident recommendation, the agent flags uncertainty rather than generating a low-confidence output silently.

## Agent 1: AI Field Consultant

**Purpose:** Augment field consultant productivity by automating pre-visit preparation, in-visit note structuring, and post-visit administrative tasks — allowing consultants to spend 100% of their on-site time coaching franchisees rather than managing paperwork.

| Dimension | Detail |
|---|---|
| **Trigger** | Scheduled field visit within 48 hours; or manual request from FC |
| **Data Inputs** | Location Health Score, open corrective actions, LMS completion data, audit history, review platform data, prior visit history, franchisee relationship notes |
| **Agent Actions** | Generate pre-visit brief (30-second synthesis of 6+ data sources); structure voice observations in real time (auto-tagging); draft corrective action plan from tagged observations; generate post-visit summary |
| **Approval Model** | Pre-visit brief: available to FC immediately (no approval required — it's an input, not an output). Corrective action plan: requires FC review and approval before dispatch to franchisee. Post-visit summary: requires FC review before submission. |
| **Trust Requirements** | Source citation on every brief data point; confidence indicator on AI-drafted corrective actions; FC can modify any auto-generated content before it is finalized; all overrides captured |
| **Success Metrics** | 8+ hours/week admin recaptured per FC; <2 hour time-to-summary submission; 75%+ AI corrective action plan adoption rate |

## Agent 2: AI Launch Coordinator

**Purpose:** Monitor new franchise openings and identify launch delay risk early enough to enable intervention before delays compound.

| Dimension | Detail |
|---|---|
| **Trigger** | New franchise agreement signed; opening date set in system |
| **Data Inputs** | Pre-opening milestone completion tracker, historical opening timelines for comparable locations, permit status, training completion data for franchisee and staff, equipment procurement records |
| **Agent Actions** | Continuous milestone completion monitoring; weekly opening risk score update; automatic escalation when critical-path milestones fall behind; draft intervention recommendations for at-risk openings |
| **Approval Model** | Monitoring and risk scoring: fully automated, no approval required. Escalation notifications: automated to designated pre-opening coordinator. Intervention recommendations: generated as drafts for human approval before franchisee communication. |
| **Trust Requirements** | Risk score with confidence level and specific at-risk milestone identification; escalation includes data justification; intervention recommendations include SOP reference and rationale |
| **Success Metrics** | 4+ week lead time on delay prediction; 80%+ of delayed openings flagged before delay becomes unrecoverable; pre-opening coordinator satisfaction with alert quality |

## Agent 3: AI Compliance Agent

**Purpose:** Automate risk-stratified audit scheduling, generate compliance risk predictions, and draft corrective action plans — converting a reactive compliance function into a proactive risk management system.

| Dimension | Detail |
|---|---|
| **Trigger** | Continuous monitoring; triggered by health score compliance dimension changes, upcoming audit deadlines, or corrective action expiration events |
| **Data Inputs** | Compliance risk score (sub-dimension of Location Health Score), audit history, corrective action backlog, training completion data, prior audit failure patterns |
| **Agent Actions** | Risk-stratified audit schedule recommendations; compliance risk alerts for locations approaching high-risk state; auto-generation of corrective action plan drafts from audit findings; corrective action expiration escalations |
| **Approval Model** | Audit schedule recommendations: presented as recommendations for compliance manager approval; schedule changes are not executed automatically. Compliance risk alerts: fully automated routing to field consultant and compliance manager. Corrective action plan drafts: generated automatically, dispatched to franchisee only after compliance manager or FC approval. |
| **Trust Requirements** | Compliance risk score with source data and confidence level; audit schedule recommendation with rationale; corrective action plan with SOP citations; all regulatory-touch actions require human approval |
| **Success Metrics** | 75%+ audit failure prediction accuracy; corrective action closure rate improvement to 85%+; compliance manager time freed for coaching vs. administration (+30%) |

## Agent 4: AI Operations Copilot

**Purpose:** Provide operational managers with an always-on intelligent assistant for portfolio-level operational decisions — surfacing anomalies, routing alerts, and answering complex operational queries about the portfolio.

| Dimension | Detail |
|---|---|
| **Trigger** | Anomaly detected in operational data stream; manager-initiated query; scheduled weekly portfolio summary |
| **Data Inputs** | Location Health Score portfolio view, corrective action pipeline, field visit history, operational benchmarking data, financial performance data |
| **Agent Actions** | Portfolio anomaly alerts; conversational query responses about portfolio operational status; weekly portfolio intelligence summary; what-if analysis ("what would happen to this location's health score if training completion improved to 80%?") |
| **Approval Model** | Anomaly alerts: fully automated, routed to appropriate owner based on alert type. Conversational responses: generated and displayed; no approval required. Portfolio summaries: auto-generated and delivered on schedule. |
| **Trust Requirements** | All conversational responses cite source data; confidence level shown for all analytical conclusions; what-if analysis clearly labeled as model projection |
| **Success Metrics** | VP Operations weekly active use; alert-to-action rate 80%+; time saved on manual reporting (target: 5+ hours/week) |

## Agent 5: AI Field Coach

**Purpose:** Provide field consultants with ongoing coaching recommendations based on their own performance patterns — identifying which coaching approaches produce the most reliable location improvements and which skill gaps to address.

| Dimension | Detail |
|---|---|
| **Trigger** | Monthly performance analysis cycle; post-visit effectiveness analysis |
| **Data Inputs** | FC visit history, health score changes at visited locations, corrective action closure rates by FC, visit quality scores, peer FC performance comparison |
| **Agent Actions** | Monthly FC performance summary with peer comparison; identification of coaching approach patterns correlated with highest outcome improvement; skill gap recommendations for FC professional development |
| **Approval Model** | Performance summaries: generated and delivered to FC and their manager. Coaching recommendations: generated as suggestions; no mandatory action. |
| **Trust Requirements** | Performance comparisons are peer-group controlled for portfolio difficulty; coaching recommendations are framed as suggestions, not performance judgments; data used for FC coaching is clearly disclosed to FCs |
| **Success Metrics** | FC performance improvement rate (improvement in visit effectiveness scores over 90-day period); FC satisfaction with coaching recommendations (survey); adoption rate of recommended coaching approach adjustments |

## Agent 6: AI Executive Intelligence Agent

| Dimension | Detail |
|---|---|

| **Success Metrics** | Executive brief open rate 80%+; board package preparation time reduction to <1 hour; executive satisfaction score 4.5/5 or higher |

## Agent 7: AI Workforce Coach

**Purpose:** Identify training gaps at the individual employee and location level and generate targeted learning recommendations that close specific operational performance gaps.

| Dimension | Detail |
|---|---|
| **Trigger** | Corrective action creation with training-related category; health score training dimension drop; new employee onboarding start; certification expiration approaching |
| **Data Inputs** | Employee training history, role definition, audit findings relevant to the employee's role, corrective action history by finding category, LMS completion and quiz performance data |
| **Agent Actions** | Individual training pathway recommendations; certification gap alerts for employees approaching expiration; onboarding completion risk flags for new hires |
| **Approval Model** | Training recommendations: delivered to store manager and franchisee as suggestions; enrollment in recommended modules requires manager or franchisee approval. Certification gap alerts: automated to manager with deadline. |
| **Trust Requirements** | Training recommendations include specific operational rationale (e.g., "Recommended because food safety training is linked to 3 of the last 4 corrective action items at this location"); employee data privacy protections applied |
| **Success Metrics** | Training completion rate improvement (target: +20% for locations using AI recommendations vs. control); certification currency rate improvement; corrective action recurrence reduction for training-linked finding categories |

## Agent 8: AI Financial Analyst

**Purpose:** Monitor financial signals across the franchise portfolio — detecting royalty anomalies, predicting franchisee financial distress, and generating financial exception reports for the finance team.

| Dimension | Detail |
|---|---|
| **Trigger** | Weekly POS reconciliation cycle; royalty calculation cycle; financial anomaly detection event |
| **Data Inputs** | POS sales data (direct integration), franchisee self-reported sales data, royalty payment history, labor cost data, historical financial patterns by location |
| **Agent Actions** | Royalty anomaly detection (statistical comparison of POS data vs. self-reported sales); franchisee financial distress prediction (payment pattern analysis); labor cost anomaly flagging; weekly financial exception report generation |
| **Approval Model** | Anomaly flags: auto-generated and delivered to finance team for review. Financial distress alerts: auto-delivered to FC and VP Operations for relationship assessment. Exception report: auto-generated weekly summary for CFO; no approval required for generation. External financial actions (invoices, formal notices) always require explicit human authorization. |
| **Trust Requirements** | Anomaly flags include the specific data comparison that triggered the flag; confidence level clearly stated; false positive rate tracked and reported; financial distress prediction includes comparable case data |
| **Success Metrics** | Royalty leakage reduction (target: 60-80% of detectable leakage identified within 90 days of POS integration); franchisee financial distress prediction accuracy (75%+ of predicted distress events confirmed within 60 days if no intervention); finance team hours saved on manual reconciliation |
